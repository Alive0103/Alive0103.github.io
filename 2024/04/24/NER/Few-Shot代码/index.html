export_model.py 处理数据并转换成ONNX格式存储

validate_onnx()

确保将PyTorch模型转换为ONNX格式后，其输出与原始PyTorch模型的输出一致

  1. 首先，它使用transformers库中的AutoTokenizer来创建一个分词器，并使用它对一些示例文本进行编码，以生成输入张量。
  2. 使用pt_model（预训练的PyTorch模型）对这些输入进行前向传播，得到预测结果。
  3. 将这些结果转换为NumPy数组，以便与ONNX模型的结果进行比较。
  4. 创建一个ONNX Runtime会话（InferenceSession），加载由onnx_path指定的ONNX模型文件。
  5. 将PyTorch模型的输入张量转换为NumPy数组，并使用ONNX模型进行推理，得到ONNX模型的输出结果。
  6. 检查ONNX模型的输出名称是否与PyTorch模型的输出名称相匹配，以确保输出的一致性。
  7. 对每个输出，检查形状和值是否与PyTorch模型的输出相匹配。如果形状或值有任何差异，则抛出异常，否则输出验证通过的信息。

ONNX格式

ONNX（Open Neural Network Exchange）是一个开放的深度学习模型表示格式，由微软和Facebook共同开发。它旨在成为深度学习框架之间的通用模型表示标准，允许用户在不同的深度学习框架之间轻松迁移模型。

ONNX格式定义了一种通用的模型表示方法，可以描述神经网络的结构、参数和操作。这种通用性使得用户可以使用一个框架训练模型，然后将其转换为ONNX格式，并在另一个框架中加载和使用它，而无需重新训练模型。

export_onnx()

将预训练的PyTorch模型转换为ONNX格式，并保存为ONNX文件

  1. with torch.no_grad():：进入一个上下文管理器，确保在此范围内的计算不会跟踪梯度，以节省内存并提高性能。
  2. model = model.to(device)：将模型移动到指定的设备（GPU或CPU）上进行推理。
  3. model.eval()：将模型设置为评估模式，这通常会关闭模型中的dropout和batch normalization等层。
  4. model.config.return_dict = True和model.config.use_cache = False：设置模型配置，指示模型返回输出字典，并禁用缓存。
  5. if not args.output_path.exists(): args.output_path.mkdir(parents=True)：检查输出路径是否存在，如果不存在则创建。
  6. dynamic_axes = {name: {0: 'batch', 1: 'sequence'} for name in chain(input_names, output_names)}：创建一个动态轴字典，用于指定输入和输出的动态维度。
  7. 生成虚拟输入数据：
    • batch_size和seq_length确定了虚拟输入数据的批次大小和序列长度。
    • dummy_input是一个包含了未知标记（[UNK]）的虚拟输入序列列表，用于生成模型输入。
    • 使用tokenizer将虚拟输入数据编码为PyTorch张量。
  8. if save_path.exists(): save_path.unlink()：检查是否已存在要保存的ONNX文件，如果存在则删除，以防止覆盖。
  9. torch.onnx.export()：使用torch.onnx.export函数将PyTorch模型转换为ONNX格式。
    • 第一个参数是模型对象。
    • 第二个参数是模型的输入数据，以元组的形式传递。
    • 第三个参数是要保存的ONNX文件路径。
    • input_names和output_names分别指定了输入和输出的名称。
    • dynamic_axes指定了动态维度的名称和位置。
    • do_constant_folding=True用于执行常量折叠优化。
    • opset_version=11指定了ONNX的操作集版本。
  10. if not os.path.exists(save_path): logger.error(f'Export Failed!')：检查ONNX文件是否成功导出，如果失败则打印错误信息。
  11. 返回保存的ONNX文件路径。

convert_example()

def convert_example(example, tokenizer, max_seq_len)

将原始样本转换为模型可以接受的格式，包括对文本进行分词和编码，以及构建起始和结束标记

  • example：原始样本，是一个包含标题、提示、内容和结果列表等字段的字典。
  • tokenizer：分词器对象，用于对文本进行分词和编码。
  • max_seq_len：最大序列长度，用于控制输入文本的长度。
  1. 使用分词器对提示和内容进行分词和编码，得到输入的编码序列。使用tokenizer()方法进行处理，设置了截断、最大长度和添加特殊标记等参数。
  2. 获取编码序列中每个字符的偏移量映射，并对其进行处理，将偏移量对齐到编码序列中的位置。
  3. 根据结果列表中的实体位置信息，将编码序列中对应位置的标记设置为起始标记和结束标记。
  4. 构建包含输入编码序列、类型编码序列、注意力掩码、起始标记和结束标记的输出列表。
  5. 对输出列表中的各项进行填充，保证它们的长度都为max_seq_len。
  6. 返回转换后的样本，作为模型的输入。

map_offset()

map_offset(ori_offset, offset_mapping)

将 ori 偏移量映射到令牌偏移量,在原始文本和令牌序列之间建立起一个映射关系，以便后续处理时能够正确地处理实体的位置信息

  1. 遍历 offset_mapping 中的每个偏移量对，即每个子词的起始和结束位置在原始文本中的偏移量。
  2. 检查 ori_offset 是否落在当前偏移量对所表示的区间内。
  3. 如果 ori_offset 落在了某个区间内，则返回该区间对应的子词在令牌序列中的索引。
  4. 如果 ori_offset 不在任何一个区间内，则返回 -1，表示无法找到对应的子词。
  • ori_offset：原始文本中的字符偏移量，即原始文本中的字符位置。
  • offset_mapping：一个包含了每个子词的起始和结束字符在原始文本中的偏移量的列表，这个列表是由分词器生成的。

model.py 模型损失计算

model_base.py 和 model.py差不多，模型加载被注释

二元交叉熵损失函数通常用于二分类任务，其中每个样本只有两个可能的类别。在自然语言处理中，序列标注任务（如命名实体识别、问答任务中的答案抽取等）通常被建模为一个二分类问题，即对每个位置（或标记）进行分类，判断该位置是否为目标实体的起始或结束位置。

因此，在命名实体识别等任务中，起始位置和结束位置的预测可以视为一个二分类问题，而二元交叉熵损失函数则是用于衡量模型的预测与真实标签之间的差异，是二分类任务中常用的损失函数之一。

具体来说，二元交叉熵损失函数将模型预测的概率与真实的标签进行比较，如果模型预测正确，则损失接近于零，否则损失较大。通过最小化损失函数，模型可以在训练过程中调整参数，以使得模型的预测更接近于真实标签，从而提高模型的性能。

因此，在起始位置和结束位置的预测任务中，使用二元交叉熵损失函数是为了使模型能够学习正确地预测实体的起始和结束位置，从而提高模型在序列标注任务中的性能。

对于每个batch，首先将数据传入模型进行前向传播，得到模型的预测结果和损失值。然后根据损失值进行反向传播，更新模型参数。

在每个epoch的训练过程中，记录并输出训练过程中的损失值和训练速度等信息。

在每个epoch结束后，进行模型的评估。评估过程包括使用验证集对模型进行评估，并计算评估指标（如精确度、召回率、F1值）。

根据评估指标的表现，选择是否保存模型。如果评估指标优于之前的最佳指标，则保存当前模型为最佳模型。

可选地进行早停策略。如果评估指标在一定的连续迭代中没有提升，则停止训练，并保存当前模型。

这段代码的逻辑比较清晰，主要包括模型的训练和评估过程，以及一些训练过程中的辅助操作，如EMA、FGM和早停策略。

predict.py 信息提取

  • 读取给定文件路径中的原始数据集文件，该文件应该是一个 JSON 格式的文件。
  • 遍历数据集中的每个样本，并针对每个样本进行信息提取。
  • 对于每个提取的结果，构造一个字典，包括当前任务的 ID、测试任务的 ID、样本 ID、文本、起始位置、结束位置和类型。
  • 将所有提取的结果保存到一个新文件中。

函数 parse_data 接受以下参数：

  • file_path：原始数据集文件的路径。
  • current_task：当前任务的 ID。
  • test_task：测试任务的 ID。
  • task_name：任务名称，用于从信息提取结果中提取对应任务的信息。
  • write_name：保存提取结果的文件路径。

在函数内部，首先读取原始数据集文件，然后遍历其中的每个样本。对于每个样本，调用信息提取函数（假设为 ie）进行提取。提取的结果是一个字典列表，包含提取的文本片段及其位置等信息。然后，将提取的结果转换为指定格式的字典，并添加到 data 列表中。最后，将整个 data 列表保存到指定的文件中。
